TYPE CONVERSION - JAVASCRIPT

Analyzing the Behavior of the stringType() Function"

            
                function stringType() {
                    let a = "coconut";
                    let b = "coconut";
                    let c = "coconut";
                    a = Number(a); // Convert 'a' to a number
                    b = String(b); // Convert 'b' to a string (although it's already a string)
                    c = Boolean(c); // Convert 'c' to a boolean
                    console.log(a, typeof a);
                    console.log(b, typeof b);
                    console.log(c, typeof c);
                }
  1. NaN and its type number: When you try to convert the string "coconut" to a number using Number(), it results in NaN (Not a Number). This is because "coconut" is not a valid number. So, the value of variable 'a' becomes NaN after the conversion.
  2. "coconut" and its type string: Variable 'b' is already a string, so converting it to a string using String() doesn't change its value. It remains "coconut".
  3. true and its type boolean: When you convert the string "coconut" to a boolean using Boolean(), any non-empty string is considered true. Therefore, the value of variable 'c' becomes true after the conversion.

                function noValue () {

  2. "" (empty string) and its type string: Variable 'b' is already an empty string, so converting it to a string using String() doesn't change its value. It remains an empty string "", and its type is string.

Analyzing the Behavior of the noAssignment() Function"

            
                function noAssignment () {
                    let a;
                    let b;
                    let c;
                    a = Number(a); // Convert 'a' to a number
                    b = String(b); // Convert 'b' to a string (although it's undefined)
                    c = Boolean(c); // Convert 'c' to a boolean (although it's undefined)
                    console.log(a, typeof a);
                    console.log(b, typeof b);
                    console.log(c, typeof c);
                }
  1. NaN and its type number: Variable 'a' is declared but not assigned a value, so it's undefined. When you convert undefined to a number using Number(), it results in NaN (Not a Number). Therefore, the value of 'a' after the conversion is NaN, and its type is number
  2. "" (empty string) and its type string: Variable 'b' is declared but not assigned a value, so it's undefined. When you convert undefined to a string using String(), it results in an empty string "". Therefore, the value of 'b' remains an empty string "", and its type is string.
  3. false and its type boolean: Variable 'c' is declared but not assigned a value, so it's undefined. When you convert undefined to a boolean using Boolean(), it results in false. Therefore, the value of 'c' remains false, and its type is boolean.
